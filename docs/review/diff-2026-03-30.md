## Critical Issues

### C-01: Observation mission fallback drops interactive input and performs no signal placement

**File:** `packages/server/src/engine/cards/base/ObservationQuickMissionCard.ts`
**Line(s):** 103-111 (in commit `8392ff7`)
**Dimension:** Logic / Quality

In the fallback branch (`setup` missing), code calls `MarkSectorSignalEffect.markByColor(...)` inside a loop, but ignores its `PlayerInput | undefined` return and then always returns `undefined`. When there are multiple sectors of the same color, `markByColor` returns `SelectOption` and does not apply a mark immediately. Ignoring that input means no signal is placed.

**Current:**
```ts
MarkSectorSignalEffect.markByColor(context.player, game, signalColor);
...
return undefined;
```

**Suggested fix:**
Use `MarkSectorSignalEffect.markByColorChain(...)` and return its `PlayerInput`, or explicitly thread continuation from `markByColor` so interactive choice is preserved.
