### W-01: The changed exchange-resource logic has no focused test coverage

**File:** `packages/client/__tests__/pages/game/GameLayout.test.tsx`
**Dimension:** Completion / Quality

The diff changes the affordability logic for `ExchangeResourcesDialog`, but the current `GameLayout` tests only verify broad layout behavior, free-action-bar visibility, and general rendering. They do not assert which exchange options are enabled/disabled for a given player state, and they do not verify the emitted `sendFreeAction` payload when an exchange is selected.

That means the new logic can regress without a test failing. This is relevant here because the code intentionally narrowed the accepted player shape and replaced generic resource lookup with explicit `CREDIT`/`ENERGY` branches.

**Suggested improvement:**
Add a focused UI test that opens the exchange dialog and asserts:

- credit-based options enable only when `resources.credit >= 2`
- energy-based options enable only when `resources.energy >= 2`
- card-based options enable only when `handSize >= 2`
- clicking an enabled option sends `EFreeAction.EXCHANGE_RESOURCES` with the expected `from`/`to` values
