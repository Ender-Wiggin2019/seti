## Critical Issues

### C-01: Scan pending input hid Spend Signal Token

**File:** `packages/client/src/features/actions/FreeActionBar.tsx`
**Dimension:** Logic Correctness
**Status:** Fixed

`FreeActionBar` returned `null` whenever any pending input existed. That blocked `SPEND_SIGNAL_TOKEN` during scan resolution, exactly when common rules allow it (`IN_RESOLUTION` + `scanActionInProgress` + signal token).

**Fix:** Pending-input mode now renders only `SPEND_SIGNAL_TOKEN` when it is legal, and otherwise hides the bar.

**Coverage:** Added `keeps spend signal token available during scan pending input` in `packages/client/__tests__/features/actions/FreeActionBar.test.tsx`.

### C-02: Mars orbit any-card reward could not be tucked

**File:** `packages/server/src/engine/effects/probe/OrbitProbeEffect.ts`
**Dimension:** Logic Correctness / Continuation Flow
**Status:** Fixed

Mars orbit resolves `any card` followed by `tuck`. The prior continuation prebuilt the tuck `SelectCard` before the selected row/deck card entered the hand, so the new card could not be tucked.

**Fix:** The any-card choice now builds the following tuck prompt lazily after the card is added to hand, while preserving immediate downstream VP resolution.

**Coverage:** Added `offers the chosen Mars orbit any-card reward in the following tuck prompt` in `packages/server/__tests__/engine/actions/Orbit.test.ts`.

## Warnings

### W-01: Scan debug event ordering placed sub-events before parent action

**File:** `packages/server/src/engine/Game.ts`
**Dimension:** Logic Correctness / Event Ordering
**Status:** Fixed

`SECTOR_MARKED` debug events could be logged before the parent `SCAN` action. Scan now logs the parent action before executing scan effects that emit sub-events.

### W-02: Storybook test setup assumed `localStorage.clear`

**File:** `packages/client/test/storybook.setup.ts`
**Dimension:** Code Quality
**Status:** Fixed

The Storybook Vitest setup failed in this environment because `window.localStorage.clear` was not always a function. The setup now guards the call.

### W-03: Seeded event merge could reorder newer live events below older history

**File:** `packages/client/src/hooks/useGameEvents.ts`
**Dimension:** Logic Correctness
**Status:** Fixed

When live events arrived before `recentEvents` from game state, merging old seeded history could move older events above newer live events. Merging now dedupes by id and sorts by event timestamp/order newest-first.
