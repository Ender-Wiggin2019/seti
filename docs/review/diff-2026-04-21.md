## Critical Issues

### C-01: Typecheck is currently broken in server tests

**File:** `packages/server/__tests__/engine/cards/alien/AnomaliesCards.test.ts`
**Line(s):** 297, 341
**Dimension:** Quality

`pnpm run typecheck` fails because `player.playedMissions` items are typed as `TCardItem` (`string | object`), but test code assumes `.id` always exists.

Current code accesses `card.id` directly:

```ts
player.playedMissions.some((card) => card.id === 'ET.17')
player.playedMissions.some((card) => card.id === 'ET.13')
```

When `card` is a string, this is invalid at type level and breaks CI.

**Suggested fix:** Normalize mission card IDs in assertions, e.g. `typeof card === 'string' ? card : card.id`.

## Warnings

### W-01: Multi-trace mission rewards overwrite input instead of chaining

**File:** `packages/server/src/engine/missions/MissionReward.ts`
**Line(s):** 57-63
**Dimension:** Logic

For trace rewards, the loop calls `promptForTraceChoice(...)` repeatedly. Each call sets `player.waitingFor`, so only the last prompt survives. This can under-award rewards when `value > 1`.

**Why it matters:** Missions that grant multiple trace placements can resolve as a single placement, violating rules/effect intent.

**Suggested fix:** Chain prompts via continuation (`onComplete`) so each resolved input schedules the next placement, rather than overwriting `waitingFor` in a loop.

### W-02: Alien trace reward input can swallow caller continuation

**File:** `packages/server/src/engine/alien/AlienState.ts`
**Line(s):** 404-413, 422-431, 466-469
**Dimension:** Logic

`createSlotSelectionInput` returns `rewardInput` immediately when slot reward is `DRAW_ALIEN_CARD`. In this branch, `onComplete` is not executed.

**Why it matters:** Any flow that depends on `onComplete` to continue (chained effects, multi-step mark flows) may stop early if the selected slot grants draw-card reward.

**Suggested fix:** Wrap reward input so that once it resolves, control returns to `onComplete` (or explicitly compose reward input + continuation).
