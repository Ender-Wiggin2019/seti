## Warnings

> Issues that may cause bugs, hurt maintainability, or deviate from architecture. Should fix.

### W-01: Card-row input path can submit non-selectable cards

**File:** `packages/client/src/pages/game/GameLayout.tsx`
**Dimension:** Logic

When `pendingInput` is card-based and *some* card-row card is selectable, `handleCardRowClick` sends input for whichever row card was clicked, without checking whether that specific card is in the selectable set.

This can emit invalid `cardIds` to the server and create avoidable error/retry loops in the UI.

Recommended: gate submission with `if (!selectableCardIds.has(card.id)) return;` before `sendInput(...)`.

---

### W-02: Direct buy buttons bypass turn/input gating used elsewhere

**File:** `packages/client/src/pages/game/GameLayout.tsx`
**Dimension:** Logic

The new `Buy From Deck` / `Buy From Row` buttons dispatch `sendFreeAction(...)` directly without checking `isMyTurn`, pending input lock, or action availability constraints. This bypasses the guardrails enforced in `ActionMenu`/`FreeActionBar` and can trigger invalid actions from the UI.

Recommended: compute disabled state from the same conditions used by free-action controls, and block click handlers when unavailable.
